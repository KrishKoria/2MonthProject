  <name>Task 1: Fix next.config.ts rewrite to exclude Next.js-owned API routes</name>
  <files>frontend/next.config.ts</files>
  <action>
Update the `rewrites()` function in `frontend/next.config.ts` so that `/api/auth/*` and
`/api/admin/*` are NOT forwarded to FastAPI and are handled by Next.js App Router instead.

Change the single rewrite rule source from `/api/:path*` to use a negative-lookahead pattern
that excludes the Next.js-owned prefixes. The Next.js rewrite `source` field supports regular
expression syntax when wrapped in parentheses with `(?!...)` negative lookahead.

Replace the existing rewrite rule:
```typescript
{
  source: "/api/:path*",
  destination: `${proxyTarget}/api/:path*`,
}
```

With this rule that excludes auth and admin prefixes:
```typescript
{
  source: "/api/((?!auth|admin).*)",
  destination: `${proxyTarget}/api/$1`,
}
```

Note: When using a capturing group in `source`, the capture is referenced as `$1` in
`destination` (not `:path*`). Confirm the pattern with the Next.js rewrites docs — the
`(?!auth|admin)` negative lookahead means any `/api/auth/...` or `/api/admin/...` request
will NOT match this rule and will fall through to App Router handlers.

The constraint from CLAUDE.md is: "Browser callers stay on /api/... paths — only the proxy
mechanism changes, not the URLs." This fix satisfies that constraint — the URLs are unchanged,
only the routing logic is updated.

Do NOT add any new routes or modify any other files in this task.

**Step 1 — Generate the Better Auth schema file.**

If the CLI is not available or fails, manually define the Better Auth core tables as shown in
step 2. The generated file will contain the Drizzle table definitions for `user`, `session`,

**Step 2 — Merge Better Auth tables into schema.ts.**

Open `frontend/src/db/schema.ts`. At the top, extend the imports from `drizzle-orm/pg-core` to
include `boolean` and `integer` (needed for `banned` and session expiry). Then add the following
Better Auth core table definitions BEFORE the existing custom app tables:

```typescript
import { boolean, integer, pgTable, text, timestamp } from "drizzle-orm/pg-core";

// Better Auth core tables — generated from auth.ts config.
// admin plugin adds: banned (boolean), banReason (text), banExpires (timestamp).
// user.additionalFields adds: role (text) — the canonical AppRole column (D-05/D-06).

export const user = pgTable("user", {
  id: text("id").primaryKey(),
  name: text("name").notNull(),
  email: text("email").notNull().unique(),
  emailVerified: boolean("email_verified").notNull(),
  image: text("image"),
  createdAt: timestamp("created_at").notNull(),
  updatedAt: timestamp("updated_at").notNull(),
  // additionalFields (D-05/D-06)
  role: text("role"),
  // admin plugin fields (D-03)
  banned: boolean("banned"),
  banReason: text("ban_reason"),
  banExpires: timestamp("ban_expires"),
});

export const session = pgTable("session", {
  id: text("id").primaryKey(),
  expiresAt: timestamp("expires_at").notNull(),
  token: text("token").notNull().unique(),
  createdAt: timestamp("created_at").notNull(),
  updatedAt: timestamp("updated_at").notNull(),
  ipAddress: text("ip_address"),
  userAgent: text("user_agent"),
  userId: text("user_id")
    .notNull()
    .references(() => user.id, { onDelete: "cascade" }),
});

export const account = pgTable("account", {
  id: text("id").primaryKey(),
  accountId: text("account_id").notNull(),
  providerId: text("provider_id").notNull(),
  userId: text("user_id")
    .notNull()
    .references(() => user.id, { onDelete: "cascade" }),
  accessToken: text("access_token"),
  refreshToken: text("refresh_token"),
  idToken: text("id_token"),
  accessTokenExpiresAt: timestamp("access_token_expires_at"),
  refreshTokenExpiresAt: timestamp("refresh_token_expires_at"),
  scope: text("scope"),
  password: text("password"),
  createdAt: timestamp("created_at").notNull(),
  updatedAt: timestamp("updated_at").notNull(),
});

export const verification = pgTable("verification", {
  id: text("id").primaryKey(),
  identifier: text("identifier").notNull(),
  value: text("value").notNull(),
  expiresAt: timestamp("expires_at").notNull(),
  createdAt: timestamp("created_at"),
  updatedAt: timestamp("updated_at"),
});

export const jwks = pgTable("jwks", {
  id: text("id").primaryKey(),
  publicKey: text("public_key").notNull(),
  privateKey: text("private_key").notNull(),
  createdAt: timestamp("created_at").notNull(),
});
```

Keep the existing `customInvitations` and `accessAuditEvents` tables and their type exports
exactly as they are. Add the Better Auth type exports at the bottom:

```typescript
export type User = typeof user.$inferSelect;
export type Session = typeof session.$inferSelect;
export type Account = typeof account.$inferSelect;
export type Verification = typeof verification.$inferSelect;
```

`jwks` enabled (omit `jwks` if not in the generated output).
